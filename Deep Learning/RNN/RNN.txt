== RNN Architecture:-
-> An RNN is designed to handle sequential data (like text, time series) by maintaining a memory of previous inputs.
-> Instead of processing inputs independently, an RNN processes them step-by-step, carrying information forward using a hidden state.
An RNN processes sequences step by step. At each time step t, it takes:
    current input --- xt
    previous hidden state --- ht-1
and produces:
    new hidden state  --- ht
    optionally an output --- yt



-> Hidden state at time t : It acts like memory of past inputs.
    ht​=f(Wh.​ht−1 ​+ Wx.​xt​ + b)

ht−1: carries past information
xt: : current input
f: activation (tanh / ReLU)

So, 
ht = past + present combined



-> Parameter sharing : Same weights are reused at every time step:
                       Wx, Wh, b stay constant

This is what makes RNNs:
    efficient (fewer parameters)
    able to handle variable-length sequences


Problems:- Vanishing / exploding gradients. Hard to learn long-term dependencies
           solved using LSTM / GRU (improved RNNs)

== Types of RNN Architecture:- 

🔹 1. One-to-One (Simple NN, not really RNN)
Used for basic tasks (like classification).
Example: Image classification

x ───► y
No sequence involved
Just standard neural network

🔹 2. One-to-Many
Single input → sequence output
Example: Image captioning / Video or Movie Recommendation

x ───► h ───► y1 ─► y2 ─► y3
One input generates multiple outputs over time

🔹 3. Many-to-One
Sequence input → single output
Example: Sentiment analysis

x1 ─► h1
x2 ─► h2
x3 ─► h3 ───► y
Final hidden state used for prediction

🔹 4. Many-to-Many (Same length)
Input and output sequences are aligned
Example: (Part of Speech) POS tagging

x1 ─► h1 ─► y1
x2 ─► h2 ─► y2
x3 ─► h3 ─► y3
Output at each time step


🔹 5. Many-to-Many (Different lengths / Seq2Seq)
Input sequence → different-length output
Example: Language translation

Encoder:   x1 ─► h1 ─► h2 ─► h3
                         │
                         ▼
Decoder:            y1 ─► y2 ─► y3 ─► y4
Encoder compresses input into context
Decoder generates output sequence



== Backpropogration:-

🔹 Setup (Many-to-One)
Input: sequence of words x1,x2,x3.....xT
Output: single label (e.g., positive/negative)
Forward flow:
x1 → h1
x2 → h2
x3 → h3 → ... → hT → y
Only the last hidden state hT is used for prediction.


🔹Forward Pass
Hidden states:
            ht = f(Wh.ht-1 + Wx.xt + b)

Output:
            y = softmax(WyhT)

Loss (for classification):
            L = CrossEntropy(y, ytrue)


🔹 Backpropagation Through Time (BPTT)
Key idea:
Since each hT depends on previous states, gradients must flow backward through all time steps.


🔹 Intuition (Sentiment Example)
Sentence: "The movie was not good"
    Early words (“movie”, “not”) influence final sentiment
    During BPTT:
        Error from final prediction flows back
        Adjusts weights so earlier words matter correctly

LSTM (Long Short-Term Memory)

🔹 Why LSTM is used
-> LSTM is designed to handle long-term dependencies in sequences.
-> In many tasks (like sentiment analysis or language modeling), early inputs matter later
-> Standard RNNs struggle to “remember” information over long sequences
-> LSTM introduces a structured memory to fix this


🔹 What RNN fails at -- 
Vanishing / Exploding Gradients:
During backpropagation, gradients shrink or grow exponentially
Result:
Early information gets forgotten (vanishing)
Training becomes unstable (exploding)

Effect in practice:
RNN forgets words far back in a sentence
Example:
“The movie was not … good” → RNN may miss “not”


🔹 Key Intuition
Forget gate → removes irrelevant info
Input gate → adds useful info
Output gate → decides what to show


🔹 Summary
RNN fails due to vanishing gradients and short memory
LSTM introduces:
cell state (long-term memory)
gates (control flow of information)
Result: learns long-range dependencies effectively
